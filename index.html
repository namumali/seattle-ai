Course Description

This course provides an in-depth examination of component security in cloud computing environments. Drawing on guidelines and best practices outlined by the Cloud Security Alliance, students gain competencies in secure software development, identity management, encryption methods, and other critical cloud security topics.

Students build expertise in assessing and mitigating vulnerabilities in cloud components like operating systems, hypervisors, orchestration engines, and more. Upon successful completion of this course, students will be prepared to sit for the Certificate of Cloud Security Knowledge certification exam.

Course Outcomes

As a result of this course, students will:

Component Security Fundamentals

Explain component security in cloud service models, component lifecycle, and supply chain management security.

Governance and Risk Management

Describe Governance and Enterprise Risk Management, Contracts and Electronic Discovery, legal, compliance and audit management frameworks in the cloud environment.

Secure Development Analysis

Analyze secure component development for a software application.

Cloud Structure Evaluation

Evaluate secure cloud structures, processes, and services.

Application Security Assessment

Evaluate a software application using secure components in the cloud.

Architecture Planning

Create a secure cloud architecture plan.

Team Project (TP)

Project Description: Secure Cloud-Based Application Design

This project provides students with an opportunity to explore the principles and practices of designing a secure cloud-based application, aligning with the Certificate of Cloud Security Knowledge (CCSK) objectives.

Project Focus: Documentation and presentation of design and security considerations for a fictional cloud-based application, rather than actual implementation.

Team Requirements:

  • Teams of 2-3 members
  • Projects must be approved by instructor
  • Choose from diverse fictional organization scenarios
  • Weekly incremental development with instructor feedback

TP01: Proposal

Initial project proposal outlining the fictional organization and security challenges to address.

30 Points

TP02: Progress Report

Mid-project progress report showing development and incorporating instructor feedback.

70 Points

TP03: Final Report

Comprehensive 7-8 page final report documenting the complete secure cloud architecture design.

70 Points

TP04: Final Presentation

15-minute presentation with 5 minutes for Q&A, showcasing project outcomes and findings.

30 Points
